## 1️⃣ GitHub Pages 部署

### 步驟 1: 建立 GitHub Repository

```bash
# 如果還沒有 git，先安裝 git：https://git-scm.com/

# 1. 在 GitHub 建立新 Repository
# - 到 https://github.com/new
# - Repository 名稱：`home-hub`
# - 選擇 Public
# - ✅ 勾選 "Add a README file"
# - 點擊 "Create repository"
```

### 步驟 2: 上傳檔案

**方法 A：使用 Git 命令（推薦）**

```bash
# 1. Clone repository 到本機
git clone https://github.com/你的用户名/home-hub.git
cd home-hub

# 2. 複製所有檔案到這個資料夾
# 複製：
# - index.html
# - module-reminders.html
# - module-inventory.html
# - module-wardrobe.html
# - module-wiki.html
# - firebase-config.js
# - 使用指南.md

# 3. 提交變更
git add .
git commit -m "Initial commit: Home Hub v2.0.0 with Firebase + Google Sheet"
git push origin main

# 完成！你的網站將在 https://你的用户名.github.io/home-hub 上線
```

**方法 B：直接上傳（網頁版）**

1. 到 https://github.com/你的用户名/home-hub
2. 點擊 "Add file" → "Upload files"
3. 拖放所有 HTML、JS 和 MD 檔案
4. 點擊 "Commit changes"

### 步驟 3: 啟用 GitHub Pages

1. 進入 Repository 設定
2. 左側選擇 "Pages"
3. 在 "Source" 選擇 "main" 分支
4. 保存

**✅ 完成！** 你的網站將在 https://你的用户名.github.io/home-hub 上線
- 可能需要等待 1-2 分鐘才能上線

### 步驟 4: 分享給家人

```
https://你的用户名.github.io/home-hub?room=MY0918
```

## 2️⃣ Google Sheet API 設置

### 前置準備：建立 Google Sheet

1. 到 https://sheets.google.com
2. 點擊 "＋" 建立新試算表
3. 命名為「家庭數位管家資料」
4. 建立 4 個分頁：
   - `提醒`
   - `庫存`
   - `衣櫥`
   - `書籤`

### 提醒分頁設置

第一列（標題行）填入：

```
id | title | type | recurrence | dueDate | amount | status | description
```

**type 選項：** financial / subscription / consumable / chore / health / other

**recurrence 選項：** once / daily / weekly / biweekly / monthly / bimonthly / quarterly / semiannual / yearly

**status 選項：** pending / completed

**範例資料：**

```
1 | 水電費 | financial | monthly | 2026-04-12 | 1200 | pending | 
2 | Netflix | subscription | monthly | 2026-04-14 | 390 | pending |
3 | 換濾芯 | consumable | quarterly | 2026-04-27 | 0 | pending |
```

### 庫存分頁設置

第一列填入：

```
id | name | category | quantity | unit | threshold | location | barcode | note
```

**category 選項：**
food / drink / frozen / clean / bath / medical / baby / kitchen / pet / office / other

### 衣櫥分頁設置

第一列填入：

```
id | name | type | size | sleeve | source | status | location | worn | note
```

**type 選項：** top / dress / pants_skirt / suit / jacket / home / accessory / other

**status 選項：** current / storage / outgrown / sold

**worn:** true 或 false

### 書籤分頁設置

第一列填入：

```
id | title | category | url | tags | note
```

**category 選項：**
recipe / health / baby / home / finance / travel / tool / memo / other

**tags:** 多個標籤用空格分隔，如「氣炸鍋 食譜」

## 4️⃣ Google Sheet 直接串接

### 獲取 Google Sheet ID

1. 打開你的 Google Sheet
2. 複製 URL 中的 ID：
   ```
   https://docs.google.com/spreadsheets/d/【這是Sheet ID】/edit
   ```

### 獲取 Google Sheets API Key

1. 到 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立新專案（或選擇現有專案）
3. 搜尋「Google Sheets API」並啟用
4. 建立 API Key：
   - 點擊「建立憑證」
   - 選擇「API 金鑰」
   - 複製金鑰
5. 限制 API Key：
   - 選擇「應用程式限制」→ 「HTTP 轉介來源」
   - 添加：
     ```
     https://你的用户名.github.io/home-hub
     https://localhost:*
     ```

### 在 App 中配置

1. 打開首頁
2. 點擊右上角 **⚙️**
3. 在「Google Sheet 串接」區域填入：
   - **Google Sheet ID**（不含引號）
   - **API Key**（不含引號）
4. 點擊 **💾 儲存配置**

### 自動同步工作流程

```
你編輯 Google Sheet
    ↓
系統每分鐘檢查一次 (60秒)
    ↓
檢測到新資料
    ↓
自動上傳到 Firebase
    ↓
所有裝置即時更新
```

**提示：** 不需要手動匯入匯出，編輯完 Sheet 後，等待最多 1 分鐘，所有變更會自動同步！

## 🆘 常見問題

### Q: 上傳到 GitHub 後沒有出現網站怎麼辦？

A: 
1. 檢查 Repository 名稱是否正確（必須是 `home-hub`）
2. 等待 2-3 分鐘讓 GitHub Pages 部署
3. 進入 Settings → Pages，確認分支是 `main`
4. 檢查主檔案名稱是否是 `index.html`

### Q: Google Sheet 資料不更新怎麼辦？

A:
1. 檢查 API Key 和 Sheet ID 是否正確
2. 確認 Sheet 有共用給你
3. 等待 1-2 分鐘（系統每分鐘同步一次）
4. 在設定頁檢查「Google Sheet 串接」狀態

### Q: Firebase 連線失敗怎麼辦？

A:
1. 檢查網路連線
2. 檢查 Firebase 配置是否正確（已預設）
3. 轉為離線模式（會自動使用 localStorage）
4. 恢復連線後自動同步

### Q: 多人同時編輯會不會衝突？

A:
- Firebase 採用「最後編輯者勝」策略
- Google Sheet 支援多人協作編輯
- 建議分工：一人編輯 Sheet，其他人用 App 快速操作
